# 旧版功能对照清单

## 功能迁移

| 验收项 | 新实现 | 验证 |
| --- | --- | --- |
| `.xlsx` / `.xls` 导入 | `domain/parser.ts`，固定 `xlsx@0.20.3` | parser 单测、E2E |
| 刷新恢复上次题库 | IndexedDB 缓存原文件与 Sheet 选择 | cache 单测、三设备 E2E |
| Sheet 多选、默认排除“透视” | `SheetSelectionModal` | 组件测试、E2E |
| 四种题型 | `QuestionCard` + 纯评分模块 | parser/answer/组件测试 |
| 一级至六级识别和筛选 | parser 与练习筛选 | 真实题库测试 |
| 四个类别筛选 | 明确关键词；普朗克归基站 | exam 单测 |
| 原等级权重 | `LEVEL_WEIGHTS` 原值迁移 | exam 单测 |
| 40/10/10/3 与 1/2/1/10 分 | `EXAM_COUNTS` / `QUESTION_SCORES` | exam 单测、E2E `/100` |
| 45 分钟、手动/超时交卷 | session effect + 清理函数 | E2E、代码审查 |
| 反馈、上一题、下一题 | `QuestionCard` / session footer | 组件测试、E2E |
| 答对立即跳题、答错停留 | 确认正确答案时同步切换索引，无延迟 timeout | E2E、代码审查 |
| 专项等级/类别/题型和统计 | `practiceCounts` 纯筛选 | UI/E2E 检查 |
| 专项中断恢复 | 仅保存 ID/索引/答案 | storage 单测 |
| 成绩页 | 实际得分/实际满分/用时/错题/正确率 | E2E |
| 错题保存、去重、统计、展示、清空 | 稳定 ID + v2 存储 | storage 单测、E2E |
| 记住等级和类别 | v2 settings | storage 单测 |
| 手动读题、无自动读题 | 仅“读题”按钮触发 | 代码审查 |

## 旧问题关闭

| 旧问题 | 处理结果 |
| --- | --- |
| 退出后计时器继续 | React effect 在离开 session 时清理 interval |
| 自动跳题跳错 | 手动切题先清 timeout；回调再次比对题目 ID |
| Excel `innerHTML` 注入 | 全部题库字段通过 React 文本节点渲染 |
| 题量不足仍显示 `/100` | 评分计算并展示实际满分，80% 实际满分通过 |
| 判断 A/B 与 √/× 不一致 | `normalizeJudgeAnswer` 统一映射 |
| 整份题目写 LocalStorage | 练习进度仅保存稳定 ID 和答案 |
| click/touchend 双绑定 | 只使用 React `onClick`/原生按钮 |
| 文档与行为不一致 | README、使用说明、部署文档按 v3 重写 |
| Pages 路径/README 地址不一致 | `base: './'`；不再写死在线 URL；构建资源自动检查 |

## 响应式与可访问性

- 单一响应式页面，不检查 UA。
- 360px 起无横向滚动；断点为 768px 和 1024px。
- 主要交互最小 44×44px，选项为键盘可操作按钮并有 `:focus-visible`。
- 使用 `100dvh`、safe-area、`hover`/`pointer` 和 `prefers-reduced-motion` 能力查询。
- viewport 允许缩放；简答文本框可随输入法页面滚动。
