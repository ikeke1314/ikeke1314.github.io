# 技能士考试刷题系统 V3.1 开发文档

本文记录当前 WPF 版本的有效架构、业务契约和 Android 迁移边界。旧 Python/PyQt 文档已经移除；实现与文档不一致时，以 `src` 和自动化测试为准。

## 1. 技术栈与目录

- `.NET 8`、WPF、MVVM：Windows 桌面界面和交互。
- `SkillExam.Core`：不依赖 WPF、SQLite 或 Excel 的领域模型与业务规则。
- `SkillExam.Infrastructure`：Excel 导入、SQLite、旧数据迁移、日志和 Windows TTS。
- `SkillExam.App`：依赖注入、页面、ViewModel 和桌面服务。
- `tests`：领域规则、持久化、真实题库、ViewModel 交互和布局回归测试。
- `exam_bank`：发布时附带的默认题库。

依赖方向必须保持为 `App -> Core + Infrastructure`、`Infrastructure -> Core`。后续 Android 版应复用业务契约和测试用例，而不是照搬 WPF ViewModel 或 SQLite 实现。

## 2. 领域模型契约

核心类型位于 `src/SkillExam.Core/Models` 和 `src/SkillExam.Core/Exam`。

### 2.1 题目

`Question` 的跨平台字段：

| 字段 | 含义 | Android 建议类型 |
| --- | --- | --- |
| `Id` | 由题型、题干、选项、答案和来源生成的稳定标识 | `String` |
| `Type` | 单选、多选、判断、简答 | `enum class` |
| `Text` | 题干 | `String` |
| `Options` | A-E 选项 | `Map<String, String>` |
| `Answer` | 题库原始答案 | `String` |
| `Levels` | 一级至六级，可多选 | `Set<SkillLevel>` |
| `Source` | 题目来源 | `String` |
| `SourceSheet` | Excel Sheet 名 | `String` |
| `Categories` | 基站、地宝、窗宝、光学组件 | `Set<QuestionCategory>` |

题目 ID 规则由 `QuestionIdentity.Create` 维护。Android 版必须使用相同规范化输入和哈希算法，否则 PC 错题、进度或未来同步数据无法关联。

### 2.2 考试规则

规则入口是 `ExamBlueprint` 和 `ExamGenerator`：

- 45 分钟，80 分及格。
- 单选 40 道 x 1 分，多选 10 道 x 2 分，判断 10 道 x 1 分，简答 3 道 x 10 分。
- 总计 63 道、100 分。
- 一级权重：一级 80%、二级 20%。
- 二至五级权重：相邻低一级 10%、本级 70%、相邻高一级 20%。
- 六级权重：五级 10%、六级 90%。
- 同一试卷按题目 ID 去重；题量不足时返回结构化缺口，不静默少出题。

抽题实现使用可注入随机种子。Android 移植时先使用固定种子复刻现有测试，再接入生产随机源。

### 2.3 答案规范化

`AnswerEvaluator` 是唯一判定入口：

- 单选提取一个 A-E 字母。
- 多选提取 A-E、去重并排序后比较。
- 判断题兼容 `A/√/对/正确/TRUE/T/是/1/YES` 和 `B/×/X/错/错误/FALSE/F/否/0/NO`。
- 简答题只执行通用文本规范化后精确比较。

不要在 UI 层复制判分逻辑。Android 应将这些边界样例直接移植为单元测试。

## 3. 题库格式

实现入口为 `ExcelQuestionBankReader`。

- 支持 `.xlsx` 和 `.xls`。
- 每个 Sheet 的第 4 行是表头，第 5 行起是数据。
- 必填列：`考题类型`、`题目`、`答案`。
- 可选列：`选项A` 至 `选项E`、`来源`、`一级` 至 `六级`。
- 空行跳过；缺少必填列或字段的行形成导入问题，不进入题库。
- 未填写来源时使用 Sheet 名；类别从来源和题干关键词中识别。

Android 首版建议把默认 Excel 在构建阶段转换为版本化 JSON 或 SQLite，运行时直接读取。这样可减小 APK 和内存压力。只有“导入用户自定义 Excel”确属首版需求时，再把 Excel 解析器作为独立适配器接入；不要让 Apache POI 等大型库进入领域层。

## 4. 持久化契约

Windows 数据目录为 `%LOCALAPPDATA%\PEGroup\SkillExam`，SQLite 初始化和迁移位于 `SqliteDatabase`。

当前数据分为：

- `settings`：JSON 设置，包括最近题库、Sheet、语音、语速、跳题延迟和减少动画。
- `error_questions` / `error_attempts`：错题快照与每次错误作答。
- `practice_sessions` / `practice_answers`：练习题序、当前位置、待重做题和答案。
- `schema_versions`：数据库迁移版本。

V3.1 的跳题延迟范围是 0-3000 毫秒，步进 100 毫秒，默认 1000 毫秒。手动切题、结束会话或取消操作必须取消待执行的自动跳题。

Android 建议使用 Room 保存同等语义的数据，DataStore 保存轻量设置。当前 SQLite 文件不是公开同步格式，不应直接复制到 Android。需要跨端迁移时，先定义带 `schemaVersion` 的 JSON 导入导出协议，并为旧版本升级写契约测试。

## 5. Android 实现建议

建议技术栈：Kotlin、Jetpack Compose、Coroutines/Flow、Room、DataStore、Android TextToSpeech。

模块边界建议：

```text
app                 Compose 页面、导航、权限和文件选择
domain              Question、考试规则、判分、练习会话
data-question-bank  默认题库和 Excel/JSON 导入适配器
data-local          Room、DataStore、迁移和备份
speech              Android TextToSpeech 适配器
```

迁移顺序：

1. 移植 `Question`、枚举、`QuestionIdentity`、`AnswerEvaluator`、`ExamBlueprint` 和 `ExamGenerator`。
2. 将 `SkillExam.Core.Tests` 的边界用例翻译为 Kotlin 测试，确保同一输入得到同一题目 ID、判分和抽题分布。
3. 实现默认题库读取和统计，再实现模拟考试、专项练习、错题本。
4. 接入 Room/DataStore，并覆盖应用升级、进程终止和数据恢复。
5. 最后接入文件选择、自定义 Excel、TTS、备份与无障碍适配。

Android 特有边界：

- 通过 Storage Access Framework 读取用户题库，不依赖真实文件路径。
- 处理进程被系统回收、旋转和后台恢复，不把会话只放在 Activity 内存中。
- TTS 生命周期随页面停止，切题时取消上一条朗读。
- 大题库解析放在后台线程，并限制一次性对象分配。
- 题库、数据库和日志不得包含用户身份、密钥或 Token。

## 6. 本地开发与发布

在 `pc-app` 目录执行：

```powershell
dotnet restore .\SkillExam.sln
dotnet test .\SkillExam.sln --configuration Release
.\scripts\publish.ps1
```

发布脚本会清空旧 `artifacts`，运行测试并生成自包含单文件便携版。最终便携目录只应包含：

```text
技能士考试刷题系统.exe
exam_bank\PE 技能士题库.xlsx
操作说明.md
开发文档.md
```

提交发布前必须确认：完整测试通过、程序版本为 3.1.0、ZIP 可读取、上述四项齐全、没有 PDB、安装器、用户数据库、日志、备份或旧 Python 文件。
